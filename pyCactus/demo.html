Using the CactusRun class

On instantiation, the CactusRun class loads the geometry file along with the time, element, and revolution data. It also parses the first line of each of the wake grid and wake node .csv files to determine the output times. Note that the module currently requires that the wake grid and node files be formatted into one .csv per timestep.

The time, element, and revolution data are stored as Pandas dataframes. In order to reduce the time for class instantiation, data is only read from files in to memory when needed.

Accessing time, element, revolution data

Time, element, and revolution data can be accessed using the appropriate variables names:

  • CactusRun.time_data
  • CactusRun.elem_data
  • CactusRun.rev_data

Each returns a Pandas dataframe.

Extracting wake data to NumPy arrays

A dataframe containing data at a particular timestep can be extracted using the function get_df_inst(). This dataframe can be interpreted using the functions wakedata_from_df() and wakegriddata_from_df(). These functions are prototyped below.

data_arrays, has_node_ids = wakedata_from_df(time)
data_arrays, grid_dims    = wakegriddata_from_df(time)

In both cases, data_arrays is a dictionary containing variables names as keys, while the values of the dictionary are NumPy arrays.

In [6]: 
# Get dataframe for t = 0.3490659
wake_df_inst = run.wakeelems.get_df_inst(time=0.3490659)
wake_df_inst.head()

# extract variables
wakedata, has_node_ids = run.wakeelems.wakedata_from_df(wake_df_inst)
